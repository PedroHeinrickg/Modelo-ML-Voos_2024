### Projeto de Machine Learning de vôos ###

O projeto de Machine Learning foi desenvolvido utilizando o dataset de simulado/histórico de voos de 2024, dispoível no kaggle: ('https://www.kaggle.com/datasets/nalisha/flight-delay-and-cancellation-data-1-million-2024/data'). O trabalho se iniciou visando desenvolver um modelo de machine learning com o objetivo principal aprimorar a logística operacional de voos aéreos, focando em duas tarefas críticas: previsão de cancelamento (classificação) e previsão do tempo de atraso (regressão).

### Procedimentos ###
O dataset foi submetido a uma limpeza inicial rigorosa, incluindo o tratamento de valores nulos e a conversão de flags para o formato binário. Análise e Engenharia de Features antes da modelagem, análises descritivas para identificar hotspots operacionais, resumindo o desempenho de cancelamentos e atrasos por cidade de origem. A etapa mais crítica foi a Engenharia de Features, que incluiu: Agregação Temporal: Criação de variáveis como time_of_day para capturar o impacto dos horários de pico. Sazonalidade: Extração do dia e semana do ano.Codificação: Aplicação de One-Hot Encoding em variáveis categóricas como origin e time_of_day. A experiência mostrou que as features criadas tiveram um impacto significativamente maior no desempenho do que a simples troca de modelos. 


### Ferramentas ###

O projeto de Machine Learning para a previsão de voos foi desenvolvido utilizando o VS Code (Visual Studio Code) como Ambiente de Desenvolvimento Integrado (IDE), o que permitiu escrever, testar e depurar o código Python de forma eficiente. O controle de versão e a colaboração foram gerenciados pelo GitHub, onde o código foi versionado, rastreado e mantido em um repositório remoto, utilizando comandos git para sincronização. O do projeto consistiu na utilização de bibliotecas do ecossistema Python. Para a manipulação e limpeza de dados, utilizamos Pandas e NumPy. O Pandas foi crucial para a leitura do arquivo CSV, tratamento de valores nulos, criação de features de tempo e congestionamento, e para todas as operações de agregação por cidade. O NumPy forneceu a base para operações numéricas de alto desempenho, como o cálculo da raiz quadrada no RMSE.O motor dos modelos preditivos foi o Scikit-learn (sklearn). Esta biblioteca forneceu as ferramentas para dividir o dataset em conjuntos de treino e teste (train_test_split) e para implementar os algoritmos de Machine Learning: Classificação (Previsão de Cancelamento): Utilizamos a Regressão Logística e a Árvore de Decisão para prever a probabilidade de um voo ser cancelado. Regressão (Previsão de Atraso): Utilizamos a Regressão Linear como modelo de baseline e o Random Forest Regressor, um algoritmo ensemble robusto, para tentar capturar a natureza não-linear do problema de atrasos. A avaliação dos modelos foi realizada utilizando as métricas do Scikit-learn, como o classification_report e a confusion_matrix (com visualização aprimorada por Seaborn e Matplotlib) para classificação, e o R² e o RMSE (calculados via r2_score e mean_squared_error) para regressão.


### Conclusões ###

Conclusões da Modelagem dividimos o projeto em dois objetivos principais: 

1. Previsão de Cancelamento (Classificação): O modelo de Regressão Logística entregou resultados impressionantes, com Acurácia de 0.9998 e Recall da Classe 1 (Cancelado) de 0.99 .Inferência Principal: Embora o desempenho seja excelente, a performance irrealista sugere um alto risco de Data Leakage. É provável que features que só são preenchidas após o cancelamento (como códigos de cancelamento ou tempos reais de partida) tenham sido usadas como preditores. Ação Recomendada: É obrigatório revalidar o modelo após remover qualquer coluna que vaze a informação do evento de cancelamento antes de qualquer implementação em produção. O valor primário deste modelo reside na sua alta interpretabilidade (Regressão Logística) e na minimização de Falsos Negativos (erros caros); 

2. Previsão do Atraso Total (Regressão): A Regressão Linear e o Random Forest Regressor foram usados para prever o tempo total de atraso em minutos. Regressão Linear Básica: Demonstrou ser inútil com R² de apenas 0.0121. A Regressão Linear Aprimorada: O R² subiu para 0.2188 ao incorporar features de hora e origem, provando o valor da engenharia de features. E a Random Forest Regressor: Não conseguiu ir além de R² de 0.2265;

Inferência Final de Regressão: O platô de desempenho em torno de R² = 0.22 indica que a limitação não é o tipo de modelo, mas sim a ausência de dados preditivos dinâmicos. O modelo não tem as informações essenciais (como volume de tráfego em tempo real ou o atraso de chegada da aeronave anterior) para explicar as causas de um atraso. A próxima etapa estratégica para que a previsão de atrasos atinja um patamar de utilidade (R² > 0.60), o foco deve ser integralmente na coleta e engenharia de features de pressão operacional e temporal, movendo-se além das informações estáticas de agendamento.
